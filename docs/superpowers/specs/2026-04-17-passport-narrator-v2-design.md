# Passport Narrator v2 — Design Spec

## Overview

Upgrade the AI Passport Narrator at `/passport/[sku]` to match the quality bar of the Fit Concierge: streaming text output, contextual suggestion chips, voice read-aloud, and a server-generated PDF download for caregiver handoff. Also fixes a latent bug where follow-up questions currently re-generate the initial summary instead of answering the specific question.

---

## 1. API Changes

### `/api/passport-narrator/route.ts` (overhaul)

**Request shape:** `POST { sku: string, question?: string }`

**When `question` is absent:** Generate the initial passport summary.

**When `question` is present:** Answer the follow-up in context. The passport data is included in every request as system context so the model can ground answers in the specific product. (This fixes the current bug where `question` is silently ignored.)

**Streaming:** Replace `anthropic.messages.create()` with `anthropic.messages.stream()` and return the same `ReadableStream` pattern used in `/api/fit-concierge/route.ts`. Content-Type: `text/plain; charset=utf-8`.

**`max_tokens`:** Raised from 350 to 500.

**System prompt additions:**
- "Explain certifications in plain language. GOTS = Global Organic Textile Standard, covers fiber origin and labor conditions. Fair Trade = ethical labor and fair wages certification."
- "Ground carbon footprint numbers in familiar references: 2.5 kg CO2e equals approximately driving a car for 10 miles."
- "Mention the Take-Back program when discussing end-of-life options."
- "Never say 'environmentally friendly'. Use 'lower-impact' and quantify the claim."

---

### `/api/passport-pdf/[sku]/route.ts` (new)

**Method:** GET

**Response:** `application/pdf` with `Content-Disposition: attachment; filename="tempo-passport-{sku}.pdf"`

**Logic:**
1. Look up `passports[sku]` — return 404 if not found.
2. Generate QR code PNG as data URI using `qrcode.toDataURL(url)` (Node-side, server only).
3. Call `renderToBuffer(<PassportPDFDocument passport={passport} qrDataUri={dataUri} />)` from `@react-pdf/renderer`.
4. Return the buffer as a streaming response with correct headers.

**Route segment config:** `export const dynamic = 'force-dynamic'` to prevent static caching.

## 2. PDF Document — `src/components/passport/PassportPDFDocument.tsx`

Pure `@react-pdf/renderer` component (no DOM, no Tailwind, no shadcn). Uses only `Document`, `Page`, `View`, `Text`, `Image`, `StyleSheet` from the renderer.

**Page 1 (A4, portrait):**
- Header bar: Tempo wordmark + "Digital Product Passport" label, dark background (`#1A1A1A`), gold accent (`#C29E5F`).
- Product identity block: product name, SKU, GTIN, issue date.
- QR code image (~90pt square) with caption "Scan to verify on tempo.style".
- Materials composition table: fiber, %, certified (yes/no), certification body.
- Certifications list with plain-language one-line explanations inline.

**Page 2 (A4, portrait):**
- Carbon footprint section: table of raw material / manufacturing / transport / end-of-life / total, each row annotated with "≈ X miles driving" equivalency.
- Care & sterilization: bulleted instructions, sterilization compatibility badge (YES/NO), protocol text if applicable.
- End-of-life + Take-Back: recyclability score, end-of-life instructions, Take-Back program callout box.
- Footer on both pages: "Generated by Tempo on {date}. For caregiver handoff." + page number.

**Brand palette:** `#1A1A1A`, `#FAFAF7`, `#C29E5F`, `#7A8B75`, `#E8DFD2`, `#5A5A5A`.

Props: `{ passport: DigitalProductPassport, qrDataUri: string }`

## 3. Component Changes — `PassportNarratorButton.tsx`

### Streaming

Replace the `fetch + .json()` pattern with the reader loop from `FitConciergeButton.tsx`:

```
const reader = res.body?.getReader()
while (true) {
  const { done, value } = await reader.read()
  // append decoded chunk to summary state
}
```

While streaming, render the partial text plus a blinking cursor `|` as a `<span className="motion-safe:animate-pulse">` sibling. Stop rendering the cursor once streaming completes.

State rename: `summary` stays; `loading` becomes the streaming-in-progress flag.

### Suggestion Chips

Rendered in a `role="group"` flex-wrap below the summary box, visible only once streaming is complete.

Four chips (pre-defined, not dynamic):
1. "What does GOTS certification guarantee?"
2. "Why Portugal?"
3. "Is this compatible with hospital laundry?"
4. "How does the Take-Back program work?"

Clicking a chip: sets `followUp` to the chip label text, then immediately calls `handleFollowUp()` (no extra "Ask" button click required). Follow-up responses also stream using the same reader pattern.

Style: same pill style as Fit Concierge chips — `border border-[#7A8B75]/40 bg-[#7A8B75]/10 text-[#7A8B75]`.

### Voice Output ("Read aloud")

New state: `voiceState: 'idle' | 'speaking' | 'paused'`

Guard: render only when `typeof window !== 'undefined' && 'speechSynthesis' in window`.

Voice selection: `speechSynthesis.getVoices()`, prefer first voice where `name` contains "Natural" and `lang` starts with "en". Fall back to first voice where `lang` starts with "en". Fall back to default.

Controls (appear as a compact button group beside the chips):
- **Play** (idle state): creates `new SpeechSynthesisUtterance(summary)`, sets voice, calls `speechSynthesis.speak()`. `utterance.onend = () => setVoiceState('idle')`.
- **Pause** (speaking state): calls `speechSynthesis.pause()`, sets state to `'paused'`.
- **Resume** (paused state): calls `speechSynthesis.resume()`, sets state to `'speaking'`.
- **Stop** (speaking or paused): calls `speechSynthesis.cancel()`, sets state to `'idle'`.

Accessibility: each button has a descriptive `aria-label`. The button group has `aria-label="Read aloud controls"`.

### PDF Download Button

A plain `<a>` element — no `fetch`, no state:

```tsx
<a
  href={`/api/passport-pdf/${sku}`}
  download={`tempo-passport-${sku}.pdf`}
  className="..."
>
  Download passport PDF
</a>
```

Rendered below the chips/voice controls, always visible once the summary is shown.

### New state variables

| Variable | Type | Purpose |
|---|---|---|
| `voiceState` | `'idle' \| 'speaking' \| 'paused'` | Track speech synthesis state |
| `chipsVisible` | implicit (derived from `!!summary && !loading`) | No new variable needed |

## 4. Packages

```bash
pnpm add @react-pdf/renderer qrcode
pnpm add -D @types/qrcode
```

`@react-pdf/renderer` uses its own internal reconciler and is compatible with React 19. It must only be imported in Node-runtime code (the API route) or in files that are never bundled for the browser. `PassportPDFDocument.tsx` is imported only from the API route.
